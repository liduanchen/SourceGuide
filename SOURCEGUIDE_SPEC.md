# SourceGuide 产品与架构文档

## 1. 项目定位

SourceGuide 是一个面向中文开发者的开源工具，用来把任意 GitHub 仓库或本地项目转换成源码学习路线。

它不是普通的 README 总结器，也不是单纯的文档生成器。它的核心价值是：

同一个仓库，不同目的的人应该有不同读法。

SourceGuide 根据用户目的生成 4 条路线：

- 我是小白路线
- 快速了解路线
- 贡献者路线
- 项目面试路线

每条路线都必须包含“如何运行这个项目”，只是讲解深度不同。

## 2. 核心目标

用户输入一个 GitHub 仓库地址或本地项目目录后，SourceGuide 自动完成：

- 获取项目源码
- 扫描项目结构
- 识别技术栈
- 找出入口文件、配置文件、核心模块、测试目录
- 判断项目如何运行
- 生成 4 条中文源码学习路线
- 生成架构说明、术语表、推荐阅读顺序和练习任务

最终产物是一组 Markdown 文档，适合阅读、收藏、发布到 GitHub、掘金、知乎或项目文档站。

## 3. 目标用户

### 小白开发者

想看懂一个开源项目，但不知道从哪里开始。

### 快速调研者

想在 10 分钟内判断一个项目是否值得深入。

### 潜在贡献者

想给项目提 issue、修 bug、提交第一个 PR。

### 求职和面试用户

想把一个开源项目变成简历项目、面试讲解素材或学习案例。

## 4. 用户入口

SourceGuide 第一版以 CLI 为主。

支持两类输入：

- GitHub 仓库地址
- 本地项目目录

用户可以选择生成全部路线，也可以只生成某一条路线。

支持参数：

- 输出语言，默认中文
- 生成路线：全部、小白、快速了解、贡献者、项目面试
- 输出目录
- 模型配置
- API base url
- 教程深度
- 是否覆盖已有文档
- 是否生成架构图
- 是否生成术语表

## 5. 输出目录结构

默认输出到项目内的 `docs/sourceguide/`。

输出文件包括：

- `README.md`：总入口，解释这个项目适合怎么学
- `01-我是小白路线.md`
- `02-快速了解路线.md`
- `03-贡献者路线.md`
- `04-项目面试路线.md`
- `architecture.md`：架构图和模块关系
- `glossary.md`：术语解释
- `exercises.md`：练习任务
- `run-guide.md`：统一运行说明，可被各路线引用
- `source-map.md`：核心文件地图

## 6. 四条路线定义

### 我是小白路线

目标：让基础一般的用户也能跑起来、看懂主流程。

必须包含：

- 这个项目是做什么的
- 适合什么基础的人学习
- 需要提前知道哪些知识
- 如何运行这个项目，必须极其详细
- 每条命令是什么意思
- 每个依赖大概有什么用
- Windows、macOS、Linux 注意事项
- 从哪个文件开始看
- 哪些文件暂时不用看
- 常见术语解释
- 常见报错排查
- 适合小白的 3 个练习任务

小白路线不允许假设用户已经懂框架、构建工具、包管理器、环境变量、测试命令。

### 快速了解路线

目标：让用户 10 分钟内判断项目价值。

必须包含：

- 项目一句话介绍
- 它解决什么问题
- 核心功能列表
- 技术栈概览
- 最短运行方式
- 项目亮点
- 项目不足或潜在风险
- 值不值得继续读
- 适合继续看的核心文件 Top 5

快速了解路线要短、直接、结论明确。

### 贡献者路线

目标：帮助用户完成第一次贡献。

必须包含：

- 如何运行项目
- 如何进入开发模式
- 如何运行测试
- 如何检查代码风格
- 项目主要模块边界
- 哪些目录适合新人切入
- 可能适合 first PR 的任务
- 提交 PR 前 checklist
- 如何定位 bug
- 如何新增一个小功能
- 项目贡献风险点

贡献者路线重点不是教学概念，而是让用户能动手改代码。

### 项目面试路线

目标：帮助用户把项目讲清楚，用于简历、面试或技术分享。

必须包含：

- 项目一句话讲法
- 项目背景和业务价值
- 如何运行项目
- 启动入口在哪里
- 启动后发生了什么
- 架构怎么讲
- 核心模块怎么讲
- 技术难点怎么讲
- 可以写进简历的表述
- 面试官可能追问的问题
- 每个问题的回答思路
- 项目可优化方向

项目面试路线要帮助用户从“我看过”变成“我讲得清楚”。

## 7. 全局生成规则

所有路线都必须包含“如何运行这个项目”。

如果 SourceGuide 无法确定运行方式，不能省略，必须输出：

- 已发现的运行线索
- 可能的启动方式
- 需要用户确认的配置
- 可能缺失的依赖
- 排查建议

所有路线都必须回答：

- 这个项目是什么
- 怎么运行
- 入口在哪里
- 我应该先看哪些文件
- 遇到问题怎么排查

禁止输出空泛内容，例如：

- “该项目结构清晰”
- “代码质量较高”
- “功能丰富”
- “适合学习”

除非后面有具体文件、模块或证据支撑。

## 8. 系统架构

SourceGuide 分为 8 个核心模块。

### CLI 入口层

负责解析用户输入、参数、输出目录和路线选择。

职责：

- 接收 GitHub URL 或本地路径
- 校验输入
- 创建任务上下文
- 调用主流程
- 展示进度和结果

### 仓库获取层

负责准备源码。

职责：

- 克隆 GitHub 仓库
- 读取本地项目
- 处理临时目录
- 识别仓库名、默认分支、项目大小
- 避免重复下载

### 文件扫描层

负责理解项目结构。

职责：

- 遍历文件树
- 过滤无关目录和文件
- 识别 README、依赖文件、入口文件、配置文件、测试目录、示例目录
- 生成项目文件地图
- 统计语言和文件类型

默认忽略：

- `.git`
- `node_modules`
- `.venv`
- `dist`
- `build`
- `coverage`
- `__pycache__`
- 图片、视频、压缩包、大型二进制文件
- minified 文件
- lock 文件可读取但不作为核心分析对象

### 技术栈识别层

负责判断项目类型。

识别依据包括：

- Python：`pyproject.toml`、`requirements.txt`、`setup.py`
- Node：`package.json`
- Go：`go.mod`
- Rust：`Cargo.toml`
- Java：`pom.xml`、`build.gradle`
- Docker：`Dockerfile`、`docker-compose`
- 前端框架：Next、Vite、React、Vue、Svelte
- 后端框架：FastAPI、Flask、Django、Express、NestJS

### 运行方式推断层

这是 SourceGuide 的关键模块。

职责：

- 从 README 中提取运行步骤
- 从依赖文件推断安装方式
- 从脚本配置中寻找启动命令
- 从 Docker 文件推断容器运行方式
- 从测试配置中推断测试命令
- 给运行方式打置信度

输出：

- 推荐运行方式
- 备用运行方式
- 环境要求
- 依赖安装方式
- 测试方式
- 常见问题

### 核心文件识别层

负责找出最值得读的文件。

判断依据：

- 入口文件
- 被大量引用的文件
- 路由、服务、核心类、核心函数
- 配置中心
- 数据模型
- 测试覆盖的模块
- 示例中使用的模块
- README 中提到的文件

输出：

- 核心文件 Top 10
- 每个文件的作用
- 推荐阅读顺序
- 文件之间的关系

### AI 分析层

负责把扫描结果转换成结构化理解。

分析任务包括：

- 项目目标总结
- 技术栈总结
- 模块职责总结
- 运行方式解释
- 架构关系推断
- 路线内容生成
- 术语解释生成
- 面试问答生成
- 练习任务生成

AI 层必须分阶段生成，避免一次性把整个仓库丢给模型。

### 文档生成层

负责把结构化内容渲染成 Markdown。

职责：

- 套用固定模板
- 生成四条路线
- 生成架构文档
- 生成术语表
- 生成练习任务
- 保持标题、顺序、风格一致
- 检查每条路线是否包含运行说明

## 9. 主流程

完整流程如下：

1. 接收输入
2. 准备仓库
3. 扫描文件树
4. 过滤无关文件
5. 识别技术栈
6. 推断运行方式
7. 识别核心文件
8. 分析 README 和配置
9. 分析核心模块
10. 生成项目知识摘要
11. 生成四条路线
12. 生成架构图和术语表
13. 校验文档完整性
14. 写入输出目录
15. 打印完成结果

## 10. MVP 功能范围

第一版必须完成：

- 支持 GitHub URL
- 支持本地目录
- 支持中文输出
- 支持生成全部四条路线
- 支持只生成单条路线
- 自动识别主流技术栈
- 自动推断运行方式
- 自动生成核心文件地图
- 自动生成 Markdown 文档
- 支持 OpenAI 兼容 API
- 支持自定义模型
- 有清晰 README
- 提供至少 2 个真实仓库生成示例

第一版暂不做：

- Web UI
- VS Code 插件
- 浏览器插件
- GitHub Action
- 多人协作
- 私有仓库授权
- PDF 导出
- 文档站部署

## 11. 二期功能

二期可以增加：

- HTML 文档站
- GitHub Action
- VS Code 插件
- 多语言输出
- 增量更新
- 私有仓库支持
- 交互式架构图
- 调用链分析
- 课程式章节拆分
- 项目评分
- 学习进度 checklist

## 12. 质量标准

生成结果必须满足：

- 每条路线都有明确目标
- 每条路线都有运行说明
- 小白路线足够详细
- 快速了解路线足够短
- 贡献者路线能指导第一次 PR
- 项目面试路线能用于表达项目经验
- 不出现明显幻觉文件
- 所有引用的文件必须来自真实扫描结果
- 不确定的内容必须标注“不确定”
- 文档语言自然，不像机器模板
- 输出能直接放进 GitHub 仓库

## 13. 项目记忆点

SourceGuide 的核心宣传语：

把任意 GitHub 仓库变成源码学习路线。

更完整的中文介绍：

SourceGuide 不只是总结仓库，而是根据你的目的生成 4 条源码学习路线：我是小白路线、快速了解路线、贡献者路线、项目面试路线。每条路线都会告诉你项目怎么运行、从哪里开始读、如何理解核心代码。

英文介绍：

Turn any GitHub repository into guided source-code learning paths.
